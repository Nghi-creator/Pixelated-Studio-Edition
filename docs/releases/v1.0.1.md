# Pixelated Studio v1.0.1

This release focuses on making the desktop engine easier to ship and recover, while hardening the path from submitted ROMs to reviewed catalog entries.

## Highlights

### Multi-runtime desktop and engine support

- Added the native Linux runtime path alongside the existing libretro runtime, including a separate native Dockerfile, native runtime lock metadata, and smoke coverage for native image builds.
- Added runtime-aware launch configuration across the desktop helper, web pairing flow, backend catalog metadata, and engine session startup.
- Added runtime switching with active-session protection so users are not moved between runtimes while a game session is still running.
- Expanded catalog/runtime readiness for additional game types, including SNES/SFC, Sega Genesis/Mega Drive, and Sega 8-bit paths.
- Improved runtime diagnostics, local vault launch errors, stream fallback behavior, camera socket support, and pairing preservation through runtime restarts.

### Admin submission and review tooling

- Added admin review pages for game submissions and catalog candidates.
- Added backend admin submission routes, candidate review actions, review-state helpers, and database migration support for submission review.
- Added candidate promotion and artifact review workflows so catalog entries can be checked before becoming playable.
- Moved more admin, catalog, profile, and player operations through the shared API/query layer for safer cache updates and clearer mutation state.

### ROM-rights and catalog hardening

- Added ROM license and hosting-rights validation for curated imports, promoted catalog candidates, and admin workflows.
- Added CI checks that reject curated ROM manifests without the required rights metadata.
- Added controlled artifact mirroring, artifact integrity validation, safer local-vault filenames, and promotion-time ROM integrity coverage.
- Added importer and generator coverage for curated, Homebrew Hub, Debian native, and API-owned catalog sources.

### Docker onboarding and release flow

- Added structured Docker diagnostics in the desktop app, including targeted guidance for missing Docker, stopped daemons, disk pressure, permission issues, and unknown failures.
- Added safe Docker install/guide links plus a Start Docker recovery path that waits for Docker Desktop and resumes engine startup.
- Added local image recovery for desktop startup: image-preparation failures now surface a `Build image & retry` path and a copyable diagnostic payload.
- Added cross-platform desktop release validation for macOS DMG, Windows NSIS, and Ubuntu AppImage artifacts.
- Added an opt-in GitHub release publishing path for validated desktop installers.

## Validation

- Desktop release validation builds and smokes the packaged app on macOS, Windows, and Ubuntu before publishing installers.
- Desktop unit and integration coverage now includes Docker diagnostics, trusted Docker startup recovery, runtime switching, companion pairing, packaged preload/renderer safety, and image-build recovery.
- Web and API tests cover admin review state, catalog candidate validation, curated ROM manifest validation, data-boundary behavior, and hosted predeploy checks.

## Desktop Artifacts

The release workflow attaches the validated native desktop installers:

- macOS `.dmg`
- Windows `.exe`
- Linux `.AppImage`
