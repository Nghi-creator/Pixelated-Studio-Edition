Studio Runtime Guide

Prerequisites

This is the Developer Studio edition of PIXELATED. It requires host virtualization to run the headless Linux engine.

  • You must have Docker Desktop installed and actively running.
  • Windows users must have WSL2 or Hyper-V enabled.
  • macOS users can install and start Docker Desktop from the Applications folder.
  • Linux users may need to start the Docker service or grant their account permission to access the Docker socket.

Current Architecture

The desktop app is the host orchestrator. It starts an isolated Docker engine that runs native Libretro emulation, captures its output, and streams gameplay to the React web player over WebRTC.

The hosted Vercel app remains the public library and account surface. For LAN multiplayer, the desktop also starts a local HTTPS companion that serves the player UI and securely proxies guest traffic to the engine without sharing the host engine token.

Exposure Control

Choose the exposure mode before initializing the engine:

  • Local only: binds the engine to this computer. Use this for solo play and development when no other device needs to connect.
  • LAN multiplayer: exposes the HTTPS companion to trusted devices on your local network. After startup, share its QR or join URL plus the short-lived invite code shown in Guest Access.
  • Initialize Engine: checks Docker, prepares the engine image, removes stale runtime state, starts the container, and waits for its health check.

Exposure cannot change while the engine is running. Only share LAN access and invite codes with people you trust.
