Studio Architecture Guide

Prerequisites

This is the Developer Studio edition of PIXELATED. It requires host virtualization to run the headless Linux engine.

  • You must have Docker Desktop installed and actively running.
  • Windows users must have WSL2 or Hyper-V enabled.

What is this architecture?

Unlike traditional emulators that rely on JavaScript WebAssembly (WASM) to run locally in a browser, PIXELATED Studio uses a true Cloud-gaming architecture (similar to NVIDIA GeForce Now or Google Stadia).

When you click "Initialize Engine", this app spins up an isolated Ubuntu container. The game runs headlessly in Linux using native C++ Libretro cores, and the gameplay is captured via Python and streamed back to this UI via WebRTC and PulseAudio.

The Benefits of Containerized Emulation

While the consumer WASM edition is great for lightweight, zero-configuration play, this Studio Edition is designed for developers, server admins, and enthusiasts looking for absolute precision:

  • Frame-Perfect Accuracy: Bypasses JavaScript garbage-collection bottlenecks entirely by running native C++ binaries.
  • Network Testing: Homebrew developers can test how their games respond to WebRTC latency and encoding compression before deploying to a commercial cloud environment.
  • Cloud Deployment: Because the engine is completely containerized, this exact Dockerfile can be dropped onto AWS, Render, or DigitalOcean with zero code changes to host a public cloud-gaming node.
